        -- Mongo Cotaner:
                -- ` docker run --name mono-name1 -p 4000:27017 -d mongo`
                        -- Here "-p 4000:27017"
                                -- "-p" repreents port
                                -- "27017" is the default port of Mongo container.
                                        -- Means if i want to connect to this mongo container inside this machine then use this port.
                                -- "4000" is the port of my actual machine. If someone from outer world is accessing this container then use this port.
        
        -- Read more about `docker log`
                --  `docker logs <container-name/ID>` --> To get the logs


        -- connect "mongo" container with "mongo-express" container
                -- `docker run -p 27017:27017 -e MONGO_INITDB_ROOT_USERNAME=admin -e MONGO_INITDB_ROOT_PASSWORD=password --name mongo-db --net mongo-network -d mongo`
                                -- "--net mongo-network"  --> tells that this is the network name if some survice wants to connect to mongoDB.
                                        -- think like what if both are running on different networks. Then we need this "--net" 
                
                -- Long commands like this can be entered as below way as well 

                -- `docker run -d \     (enter as "\" won't let it run and exit)

        -- Docker Volumes:
                -- Generally datat from container does not persist means gets removed as we exit container.
                -- But datat stored by container in volumes does not go away. 
                        -- We do not know the path or id where this datat is stored as ID is big and confusing.

                -- Here we want to do:
                        -- provide a name for a volume or space. 
                        -- store the datat of "mongo" into that volume.
                        -- But datat from "express-mongo" not needed as it is there to just run the code. 

                        -- Datat cleanup is best when we execute some code as it does not infect whole system or datat if code is corrupted.
                        -- this defect code will wipe out with Container exit.

                -- `mongo-data:/data/db`:
                        -- "mongo-data" is the volume name 
                        -- "/data/db" is the container path (for mongodb, different for postgres).
                                -- Whenever a new container of mongodb is deployed or postgres.
                                -- there is a specific path, only which these container or dbs store the values.
                                -- this path and volume is permanent, regardless of container running or exit.

        
        -- `docker-compose -f docker-compose.yaml down"
                -- remove the compose docker instance and remove all containers
